# Work Items

This directory holds implementation work that traces back to canonical requirements.
Canonical work-item artifacts are authored in `.json`, and `workbench` now
points at those canonical JSON artifacts directly.

## Domain Areas

- [`quic/`](quic/README.md): QUIC work items

## Workboard

- [`README.md`](README.md) is the workboard file configured by [`.workbench/config.json`](../../.workbench/config.json).

Work-item files should link the requirements they address, the design they use, and the verification artifacts they expect.
