REQ-QUIC-CRT-0112 - REQ-QUIC-CRT-0113 - REQ-QUIC-CRT-0114  ## Design Inputs  - ARC-QUIC-CRT-0012  ## Planned Changes  - Extend the server-role handshake continuation so it formats one local Certificate message from a narrow raw-DER leaf certificate seam only after the supported ClientHello has been accepted, the local ServerHello has been published, local EncryptedExtensions has been published, and Handshake keys are available. - Reuse the existing bridge-owned transcript seam to append the local Certificate bytes immediately after the local EncryptedExtensions and expose the bytes through the existing Handshake CryptoDataAvailable path with deterministic offset semantics. - Keep server-role peer transport-parameter staging from ClientHello intact, keep server-role commit unavailable, and reject premature, repeated, conflicting, malformed, or unsupported local Certificate progression deterministically through the existing fatal/update path. - Add focused requirement-home tests for the positive Certificate continuation, early emission prevention, local leaf-certificate availability gating, deterministic failure cases, repeated/conflicting progression rejection, transcript append and offset semantics, and the continued absence of server-role commit. - Do not add CertificateVerify, server Finished, endpoint-host wiring, harness changes, or interop-runner handshake enablement in this slice.  ## Out of Scope  - server CertificateVerify, server Finished, client-certificate authentication, trust-store policy, hostname validation, certificate-path validation, revocation, 0-RTT, 1-RTT, key update, endpoint-host wiring, harness changes, interop-runner handshake support, or any native TLS / QUIC fallback.  ## Verification Plan  Run the updated CRT requirement-home tests that cover the positive Certificate continuation path, deterministic malformed or unsupported local certificate handling, outbound local Certificate availability, offset semantics, local certificate availability gating, transcript append behavior, and repeated or conflicting progression rejection, then run the relevant solution build and targeted test pass for the main library and tests project.  ## Completion Notes  The resulting slice is intentionally smaller than full server handshake support.
