## Planned Changes

- Multi-tag the existing protected-packet write proof so the stream-level blocked-signal requirements point at a real outbound STREAM_DATA_BLOCKED packet.
- Keep negative runtime coverage proving an in-flight ack-eliciting packet suppresses duplicate blocked-signal emission while the write still fails honestly for lack of credit.
- Confirm the protected advisory packet remains retained by the sender/recovery ledger and can be retransmitted after loss.
- Add a permanent BenchmarkDotNet case for the stream-state helper branch that publishes `STREAM_DATA_BLOCKED`.
- Add focused architecture and verification artifacts for the bounded `STREAM_DATA_BLOCKED` runtime emission slice.

## Out of Scope

- Waiting asynchronously for later stream-level flow-control credit.
- Receiver-side flow-control autotuning.
- Changing DATA_BLOCKED or STREAMS_BLOCKED behavior.
- Public API surface changes.
- 0-RTT.
- Key update.
- Migration behavior.
- A general stream scheduler.
