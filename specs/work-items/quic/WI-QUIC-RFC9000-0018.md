# WI-QUIC-RFC9000-0018 - QUIC RFC 9000 Application ACK Piggyback Policy Work Item

## Summary

Implement and prove the active-path 1-RTT Application Data ACK piggyback floor for ordinary ack-eliciting sends while preserving ACK-only feedback-loop protection.

## Requirements Addressed

- REQ-QUIC-RFC9000-S13P2P1-0009
- REQ-QUIC-RFC9000-S13P2P1-0010
- REQ-QUIC-RFC9000-S13P2P1-0012

## Design Inputs

- ARC-QUIC-RFC9000-0018

## Planned Changes

- Add an active-path 1-RTT application send composition step that prepends a pending Application Data ACK frame to ordinary ack-eliciting payloads.
- Keep retransmission planning tied to the original retransmittable plaintext payload rather than the ACK-prefixed packet payload.
- Make ACK-generation state suppress duplicate ACK-only or repeated piggyback triggers after any ACK frame covers the latest ack-eliciting receipt.
- Keep ACK-only sends non-ack-eliciting and non-retransmittable, with no injected PING or STREAM frames.
- Add direct requirement-home positive, negative, edge, and deterministic fuzz/property coverage for S13P2P1-0009, S13P2P1-0010, and S13P2P1-0012.
- Add a permanent BenchmarkDotNet suite for the ACK piggyback policy checks and preserve local benchmark evidence.

## Out of Scope

- Initial and Handshake packet-builder ACK piggyback policy.
- Candidate-path application send composition.
- A general delayed-ACK timer policy.
- Changing the existing receive path's immediate ACK-only send branch.
- Endpoint interop success classification.
- Public API behavior.

## Verification Plan

Run the focused RFC 9000 ACK piggyback requirement-home tests, run the ACK piggyback policy benchmark suite with BenchmarkDotNet Dry and Short jobs, render/check the touched SpecTrace artifacts, validate JSON, run `git diff --check`, and preserve evidence under `artifacts/verification/20260425-rfc9000-ack-piggyback-policy/` and `artifacts/benchmark-baseline/20260425-rfc9000-ack-piggyback-policy/`.

## Completion Notes
