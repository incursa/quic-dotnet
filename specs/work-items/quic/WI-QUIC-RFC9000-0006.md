## Summary

Close the helper-backed RFC 9000 Section 14.3 DPLPMTUD subset by adding internal per-path probe tracking, PADDING probe sizing, BASE_PLPMTU initialization, and per-address-pair MPS isolation.

## Planned Changes

- Add an internal DPLPMTUD state helper keyed by connection path identity.
- Track PMTU probe packet numbers until acknowledgment or loss.
- Promote per-path MPS only on acknowledged probes and keep losses non-promoting.
- Compute PADDING bytes for padding-based probes while requiring ack-eliciting payload bytes.
- Replace empty S14P3 requirement homes with positive, negative, edge, and fuzz coverage.
- Add a permanent BenchmarkDotNet suite for the helper's hot paths.

## Out of Scope

- Automatic PMTU probe scheduling.
- Packet-builder or UDP socket integration.
- Operating-system ICMP ingress.
- Alternative-path search.
- Public API support claims.
