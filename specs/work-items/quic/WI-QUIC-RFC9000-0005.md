## Planned Changes

- Add or preserve candidate-path probing for new local addresses before migration promotion.
- Keep the active-path recovery reset helper tied to the migration path so congestion, RTT, and ECN validation restart from initial values on a new path.
- Retain the port-only preservation branch so a port-only change can keep the prior recovery snapshot and ECN validation history.
- Add requirement-home tests that prove new-local-address probing, deferred promotion, migration reset, port-only retention, and ECN revalidation on the migrated path.

## Out of Scope

- Generalized multipath scheduling or any broader route-selection policy beyond one active path plus candidates.
- Changing unrelated handshake, stream, or endpoint-binding behavior.

## Verification Plan

Run the RFC 9000 requirement-home tests for the Section 9.1 and 9.2 migration slice, then inspect the runtime trace and recovery snapshot evidence for the reset and port-only retention branches.
