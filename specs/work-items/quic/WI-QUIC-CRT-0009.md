do not move orchestration into the interop harness, add a native TLS fallback, or widen the slice to trust-store integration, hostname validation, certificate-path building, revocation, 0-RTT, 1-RTT, key update, or server-role handshake crypto.  ## Out of Scope  - Certificate trust, hostname validation, identity authentication, certificate-path validation, revocation, 0-RTT, 1-RTT, key update, server-role handshake crypto, or any native TLS / QUIC fallback.  ## Verification Plan  Run the updated CRT and RFC9001 requirement-home tests that cover the explicit acceptance seam, the pinned-fingerprint positive and negative paths, and the tightened commit gate, then run the relevant solution build/test pass for the main library and tests project.  ## Completion Notes  The resulting slice is intentionally narrower than full handshake support.
