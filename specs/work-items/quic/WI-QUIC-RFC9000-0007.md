# WI-QUIC-RFC9000-0007 - QUIC RFC 9000 STREAMS_BLOCKED Runtime Emission Work Item

## Summary

Emit bounded protected `STREAMS_BLOCKED` advisory packets when active outbound stream opens are blocked by the peer's current stream limit while keeping the open request pending until real `MAX_STREAMS` growth.

## Planned Changes

- Refine the API pending-open requirement so advisory `STREAMS_BLOCKED` signaling no longer conflicts with the pending-open completion contract.
- Add a runtime `STREAMS_BLOCKED` payload builder and guarded send path beside the existing blocked-signal send helpers.
- Attempt advisory emission when a queued outbound open is blocked by the peer stream limit, while preserving the pending request.
- Suppress duplicate advisory emission while ack-eliciting packets or retransmissions are already outstanding.
- Add positive and negative requirement-home tests that open the protected packet and verify the STREAMS_BLOCKED direction and Maximum Streams value.
- Add permanent BenchmarkDotNet coverage for STREAMS_BLOCKED frame parse and format hot paths.

## Out of Scope

- Completing a blocked open without real `MAX_STREAMS` growth.
- Public API surface changes.
- A general stream scheduler.
- 0-RTT.
- Key update.
- Migration behavior.
- Broader stream-management parity.
