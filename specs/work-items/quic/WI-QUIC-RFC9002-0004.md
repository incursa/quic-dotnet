## Planned Changes

- Reuse the packet-space recovery ledger as the single PTO timer source of truth instead of introducing a parallel timer store.
- Reset PTO backoff on ack-eliciting send, acknowledgment, and key discard while preserving the current zero-backoff floor.
- Gate pre-confirmation client and server PTO arming on actual validation state rather than on test-only assumptions.
- Keep PTO expiration as scheduler state only, leaving packet-loss marking, probe composition, retry reset, and migration switching to separate slices.

## Out of Scope

- Probe composition or packet selection after PTO expiry.
- Retry reset behavior.
- Migration path switching.
- Treating PTO expiry as a packet-loss declaration.

## Verification Plan

Run the targeted RFC 9002 requirement-home tests and inspect the recovery-timing, send-runtime, and validation seams that govern PTO arming, restart, backoff reset, and pre-confirmation gating.

## Completion Notes

The PTO timer discipline slice is proven by the targeted requirement-home tests and the runtime helper seams already present in the repository. PTO expiry remains scheduler state only, and the probe, retry, and migration follow-ons stay separate.
