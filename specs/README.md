# Specs

This folder is the landing zone for future RFC-derived requirement artifacts for `Incursa.Quic`.

## How To Use This Area

- Start with [`requirements/REQUIREMENT-GAPS.md`](requirements/REQUIREMENT-GAPS.md) when behavior is not yet fully defined.
- Keep one specification per file and group related requirement clauses by stable technical concern.
- Keep architecture, work items, verification, tests, and benchmarks separate from requirement text.
- Use [`../docs/requirements-workflow.md`](../docs/requirements-workflow.md) for the recommended order of operations.
