# QUIC Requirement Gaps

This ledger tracks open questions, ambiguities, and follow-up decisions for QUIC requirement work.

## Open Gaps
- `interop-harness` now has a thin endpoint-host shell around the library runtime, and the managed client/listener host path already owns honest Initial/DCID bootstrap plus server Initial-response emission; the harness `handshake` testcase now dispatches into that managed path, the narrow transfer testcase now dispatches through the managed active-phase path, and the child-process-only `retry` testcase now dispatches through the managed one-Retry path under `REQ-QUIC-INT-0012`. The main library remains the owner of the connection runtime, TLS-facing transport bridge, sender/recovery ownership, and diagnostics hooks.
- `9001-05-handshake-orchestration` is no longer one executable blocker lane. The managed handshake floor is now green through server/client `Finished` proof, peer transport-parameter commit gating, server-role 1-RTT publication, client-role 1-RTT readiness, retry bootstrap replay, client/server certificate-policy floors, resumption-ticket ownership, dormant 0-RTT prerequisite capture, the first bounded client 0-RTT send attempt, early-data rejection cleanup, and the first client key-phase follow-ons under `REQ-QUIC-CRT-0103` through `REQ-QUIC-CRT-0145`. Keep this gap open only as an umbrella for handshake-adjacent follow-ons that are already traced elsewhere: `HANDSHAKE_DONE` retransmission belongs with `9000-19-retransmission-and-frame-reliability`, broader RFC 9001 key-update behavior remains under `9001-02-security-and-registry`, and interop widening remains under the `REQ-QUIC-INT-*` / public API slices rather than this umbrella gap.
- `9002-06-key-discard-lifecycle` is partially reusable at the helper layer. The repo now has an explicit TLS bridge and connection-runtime discard path for Initial/Handshake key cleanup plus the supported handshake floor, but the RFC 9002 S6P4 0-RTT rejection and secret-discard timing clauses remain blocked until the repo grows fuller 0-RTT state ownership and full 1-RTT key-update mechanics.
- `9002-06-appendix-b-constants-and-examples` is split into a helper-backed executable subset and a deferred remainder. The helper-backed `SBP1`, `SBP2-0002`, `SBP2-0004`, `SBP2-0005`, `SBP3`, `SBP4`, `SBP5`, `SBP6`, `SBP7`, and `SBP8` clauses can move now, while `REQ-QUIC-RFC9002-SBP2-0001`, `REQ-QUIC-RFC9002-SBP2-0003`, and `REQ-QUIC-RFC9002-SBP9-0001` through `REQ-QUIC-RFC9002-SBP9-0003` remain blocked until the repo has sender/runtime PMTU accounting and connection-owned key-discard cleanup. The retained `REQ-QUIC-RFC9002-SBP9-0003` / `REQ-QUIC-RFC9002-SAP11-0003` overlap stays under manual review.
- `9000-19-retransmission-and-frame-reliability` is now executable only as the residual flow-control loss-repair lane (`flow-control-loss-repair-proof-topoff` for `REQ-QUIC-RFC9000-S13P3-0018`, `REQ-QUIC-RFC9000-S13P3-0019`, and `REQ-QUIC-RFC9000-S13P3-0024`). The current repository already owns helper/runtime primitives for ACK handling, path-challenge generation, retransmission retention, congestion accounting, stream bookkeeping, and sender/recovery state. The connection-close suppression slice is now closed by requirement-home proof coverage for `REQ-QUIC-RFC9000-S13P3-0014`, and the path/CID/handshake-completion control-frame slice is closed by requirement-home proof coverage for `REQ-QUIC-RFC9000-S13P3-0028` through `REQ-QUIC-RFC9000-S13P3-0033`.
- `9000-02-stream-state` is no longer one monolithic live-stream blocker. The supported active-loopback runtime already owns stream entry plus one-shot `RESET_STREAM` / `STOP_SENDING` actions, and terminal-state send guards plus `STOP_SENDING` admission (`REQ-QUIC-RFC9000-S3P3-0001`, `REQ-QUIC-RFC9000-S3P3-0002`, and `REQ-QUIC-RFC9000-S3P3-0005`) are closed. Bidirectional composition (`REQ-QUIC-RFC9000-S3P4-0001`, `REQ-QUIC-RFC9000-S3P4-0002`, and `REQ-QUIC-RFC9000-S3P4-0003`) is now closed by requirement-home proof coverage; keep the gap open only for the remaining live-stream follow-ons tracked elsewhere.
- `9000-03-flow-control` is no longer just a helper-layer gap. `QuicConnectionStreamState` already computes fresh `MAX_DATA`, `MAX_STREAM_DATA`, and blocked-frame payloads on read/reset/write-pressure paths, and `QuicConnectionRuntime` now publishes fresh credit on read/reset paths while also queueing the blocked-control packet on blocked writes when no ack-eliciting packets are in flight. Treat the remaining work as two lanes: receiver-contract proof (`REQ-QUIC-RFC9000-S4P1-0001` through `REQ-QUIC-RFC9000-S4P2-0005`) and blocked/credit-loss repair (`REQ-QUIC-RFC9000-S4P1-0015`, `REQ-QUIC-RFC9000-S13P3-0018`, `REQ-QUIC-RFC9000-S13P3-0019`, and `REQ-QUIC-RFC9000-S13P3-0024`). Optional autotuning remains separate from the required runtime publication floor.
- `9000-11-migration-core` is mostly blocked by the absence of the broader path-selection, packet-sending, and ECN/timer surfaces. The helper-backed anti-amplification slice now closes `REQ-QUIC-RFC9000-S9P3P1-0001`, and the runtime path-state/migration orchestration slice now covers active/candidate path ownership, address-change classification, and promotion rules, but the remaining migration work is now split into two executable S9P4 lanes: recovery-reset handling for `REQ-QUIC-RFC9000-S9P4-0001`, `REQ-QUIC-RFC9000-S9P4-0002`, `REQ-QUIC-RFC9000-S9P4-0003`, and `REQ-QUIC-RFC9000-S9P4-0005`; and cross-path ACK coverage for `REQ-QUIC-RFC9000-S9P4-0004` and `REQ-QUIC-RFC9000-S9P4-0006`. The conservative timer discipline slice for `REQ-QUIC-RFC9000-S9P4-0008` through `REQ-QUIC-RFC9000-S9P4-0011` is closed by requirement-home proof coverage. The remaining S9/S9P1/S9P2/S9P3 requirements still need a richer sender and ECN surface before they can be closed cleanly.
- `9000-14-stateless-reset` is partially reusable at the helper layer. The repo now closes the helper-backed packet-layout, token-generation, amplification-sizing, 38-bit visible-prefix, matched-token drain/no-send clauses, the endpoint ingress/emission proof for `REQ-QUIC-RFC9000-S10P3-0001` and `REQ-QUIC-RFC9000-S10P3-0015`, the token-retirement invalidation and remote-address-scoped matching slices, and the reset-send limiting slice, but `REQ-QUIC-RFC9000-S10P3-0029` remains deferred because the runtime is still version-1-only and does not yet own a multi-version reset-generation proof.
- `9001-02-security-and-registry` is partially reusable at the helper layer. The helper-backed ceiling for the chunk closes `REQ-QUIC-RFC9001-S6-0002`, `REQ-QUIC-RFC9001-S8-0001`, and `REQ-QUIC-RFC9001-S10-0001` through `REQ-QUIC-RFC9001-S10-0003`. The bridge now has executable seam coverage for carrying and authenticating transport parameters plus routing TLS KeyUpdate violations into explicit connection-close handling. `REQ-QUIC-RFC9001-S6-0004` is now closed by requirement-home proof coverage, and `REQ-QUIC-RFC9001-S6-0005` is queued as the narrow `rfc9001-key-phase-detection-edge-coverage` follow-on so the managed key-phase floor can finish edge coverage without waiting for the deferred TLS KeyUpdate surface. Keep `REQ-QUIC-RFC9001-S6-0009` and `REQ-QUIC-RFC9001-S6-0010` deferred until a real TLS message-processing surface exists. `REQ-QUIC-RFC9001-S7-0001` and `REQ-QUIC-RFC9001-S9-0001` remain deferred because they are policy/document-scope items rather than executable helper behavior.

## Closed Gaps

- `retransmission-scheduler-tail` is closed. Requirement-home proof coverage now shows that `REQ-QUIC-RFC9000-S13P3-0010`, `REQ-QUIC-RFC9000-S13P3-0011`, `REQ-QUIC-RFC9000-S13P3-0013`, `REQ-QUIC-RFC9000-S13P3-0026`, and `REQ-QUIC-RFC9000-S13P3-0033` keep ACK generation, RESET_STREAM retransmission, STOP_SENDING retransmission, PATH_CHALLENGE scheduling, and HANDSHAKE_DONE repair linked to their own requirement homes. `flow-control-loss-repair-proof-topoff` is closed as well; requirement-home proof coverage now shows that `REQ-QUIC-RFC9000-S13P3-0018`, `REQ-QUIC-RFC9000-S13P3-0019`, and `REQ-QUIC-RFC9000-S13P3-0024` keep MAX_STREAM_DATA repair, post-close suppression, and blocked-signal retry linked to their own requirement homes.
- `path-cid-handshake-completion-control-frames` is closed. The repository now has requirement-home proof coverage for the path-validation response, connection-id lifecycle, NEW_TOKEN, PING/PADDING, and HANDSHAKE_DONE control-frame reliability slice across `REQ-QUIC-RFC9000-S13P3-0028` through `REQ-QUIC-RFC9000-S13P3-0033`.

- `connection-close-suppression` is closed. Requirement-home proof coverage now shows that `REQ-QUIC-RFC9000-S13P3-0014` keeps close signals outside the retransmission scheduler, so packet loss does not requeue `CONNECTION_CLOSE` datagrams for repair.

- `path-validation-timer-discipline` is closed. Requirement-home proof coverage now shows that `REQ-QUIC-RFC9000-S9P4-0008` through `REQ-QUIC-RFC9000-S9P4-0011` keep the `PATH_CHALLENGE` timer separate, retry on timeout, and avoid aggressive timer selection.

- `path-validation-probe-loss-exception` is closed. Requirement-home proof coverage now shows that `REQ-QUIC-RFC9000-S9P4-0007` keeps probe-packet loss independent at the recovery boundary, while non-probe loss still reduces the congestion controller normally.

- `bidirectional-stream-composition` is closed. Requirement-home proof coverage now shows that bidirectional streams are composed of sending and receiving parts, composite stream-state representations are allowed, and closed-state transitions remain acknowledgment-gated under `REQ-QUIC-RFC9000-S3P4-0001` through `REQ-QUIC-RFC9000-S3P4-0003`.

- `public-api-stream-entry-slice` is closed. The repository now owns the promoted stream-entry and supporting stream-capacity surface on the supported active loopback path, including `QuicConnection.AcceptInboundStreamAsync(...)`, `QuicConnection.OpenOutboundStreamAsync(...)`, `QuicStreamCapacityChangedArgs`, `QuicConnectionOptions.StreamCapacityCallback`, the supported `QuicStream` write/completion lane, the narrow `RESET_STREAM` / `STOP_SENDING` abort pair, and the bounded pending/open-resume lane. The remaining trace now lives under `REQ-QUIC-API-0014`, `ARC-QUIC-API-0005`, `WI-QUIC-API-0005`, and `VER-QUIC-API-0005`.
- `9000-02-stream-state` read-side interruption and reset-disposition behavior is closed. Requirement-home proof coverage now covers `REQ-QUIC-RFC9000-S3P2-0022` and `REQ-QUIC-RFC9000-S3P2-0023` on the supported reset-receive path, including the default reset-disposition branch and the suppressed-reset branch.
- `post-handshake-stream-child-process-sequencing` is closed. The managed interop harness path now dispatches `post-handshake-stream` after handshake completion, the client opens the first application stream, and the server accepts it on the real harness path. The remaining trace now lives under `REQ-QUIC-INT-0011`, `ARC-QUIC-INT-0004`, `WI-QUIC-INT-0004`, and `VER-QUIC-INT-0004`.
- `9000-13-idle-and-close` is closed for the runtime close/drain/lifetime slice. The runtime now enters Closing on local immediate close, replies to attributed packets while Closing, transitions to Draining on peer `CONNECTION_CLOSE`, preserves the terminal deadline when Draining follows Closing, and discards connection state when the close or drain lifetime expires. The remaining trace now lives under `REQ-QUIC-CRT-0023`, `REQ-QUIC-CRT-0025`, `REQ-QUIC-CRT-0026`, `REQ-QUIC-CRT-0030`, `ARC-QUIC-CRT-0004`, `WI-QUIC-CRT-0004`, and `VER-QUIC-CRT-0004`.
- `standalone-local-role-preflight` is closed. The repository now exposes a repo-local preflight planner plus localhost smoke tests that separate request parsing, qlog selection, endpoint resolution, and transfer-path mapping from the direct client/server execution lane under `REQ-QUIC-INT-0014`. The external helper for peer-implementation runs remains under `REQ-QUIC-INT-0013`.
- `interop-runner-local-execution-helper` is closed. The repo-local helper now builds the harness image, invokes the external `quic-interop-runner` locally through an image-replacement slot, and captures JSON, Markdown, stderr, build-log, and log-tree artifacts under repo-local `artifacts/interop-runner/<timestamp>-<slot>/` without changing the runner repository. The helper now also supports split client/server local runs against established peer implementations while the harness tolerates the runner's empty server-side `REQUESTS` list on supported server-role paths. The remaining trace now lives under `REQ-QUIC-INT-0013`, `ARC-QUIC-INT-0006`, `WI-QUIC-INT-0006`, and `VER-QUIC-INT-0006`.
- `combined-stream-abort-both` is closed. The supported bidirectional active-loopback path now composes the existing `RESET_STREAM` and `STOP_SENDING` lanes to honor `QuicAbortDirection.Both` on `QuicStream`, while broader stream-management parity remains deferred. The remaining trace now lives under `REQ-QUIC-API-0006`, `ARC-QUIC-API-0001`, `WI-QUIC-API-0001`, and `VER-QUIC-API-0001`.
- `client-early-data-disposition-boundary` is closed. - `host-facing-qlog-capture-serialization` is closed. The remaining trace now lives under `WI-QUIC-CRT-0033`, `REQ-QUIC-CRT-0136`, and `VER-QUIC-CRT-0033`.
- `post-handshake-ticket-update-seam` is closed. The library now has the narrow internal post-Finished ticket-bearing TLS update seam tracked separately as `REQ-QUIC-CRT-0128`, `ARC-QUIC-CRT-0026`, `WI-QUIC-CRT-0026`, and `VER-QUIC-CRT-0026`; it surfaces opaque ticket bytes internally after Finished without ticket persistence, replay ownership, PSK derivation, resumed-handshake support, 0-RTT packet support, anti-replay, key update, transfer, retry, or any public API widening. - `post-handshake-ticket-wire-ingress` is closed. The client-side 1-RTT `NewSessionTicket` ingress follow-on is tracked separately as `REQ-QUIC-CRT-0129`, `ARC-QUIC-CRT-0027`, `WI-QUIC-CRT-0027`, and `VER-QUIC-CRT-0027`; it consumes a valid TLS 1.3 `NewSessionTicket` after Finished, extracts opaque ticket bytes through the existing internal seam from `REQ-QUIC-CRT-0128`, keeps pre-Finished and server-role ingress rejected, and keeps unsupported 1-RTT post-handshake TLS messages ignored rather than broadening the parser into a general post-handshake engine. - `0rtt-resumption-ticket-ownership` is closed. - `clienthello-psk-attempt-path` is closed.
